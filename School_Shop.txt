Проект 4 - Маркетплейс объявлений

📋 Контекст
Студенты работают в тройке. Каждая команда строит полноценный маркетплейс — площадку где люди могут публиковать, редактировать, удалять объявления, искать и фильтровать их. Самый сложный проект курса — полный CRUD.

Стек: HTML/CSS/JS + JSONBin.io + Vercel

Время: 3 урока × 4 часа = 12 часов

Команда: 3 человека


🎯 Бриф от заказчика

«Нам нужен сайт объявлений для школьного маркетплейса. Ученики продают/обменивают вещи между собой: учебники, гаджеты, одежду. Нужно создавать объявления, искать по категориям, писать продавцу.»



✅ Функциональные требования
Обязательные (MVP) — полный CRUD

[ ]  Create: форма создания объявления (название, описание, цена, категория, контакт)
[ ]  Read: список всех объявлений + страница одного объявления
[ ]  Update: редактирование своего объявления
[ ]  Delete: удаление своего объявления
[ ]  Категории: учебники, электроника, одежда, другое
[ ]  Поиск по названию
[ ]  Фильтрация по категории
[ ]  Простая авторизация: никнейм в localStorage (чтобы редактировать/удалять только свои)

Продвинутые (бонус)

[ ]  Загрузка фото (base64 в JSONBin или placeholder по категории)
[ ]  Сортировка: по цене, по дате
[ ]  Пагинация (по 10 объявлений на страницу)
[ ]  Счётчик просмотров для каждого объявления
[ ]  Избранное (localStorage)


🧠 Инженерные вопросы для обсуждения
1. Как организовать CRUD?

Что такое CRUD? Почему 80% всех приложений — это CRUD?
Create: как добавить элемент в массив в JSONBin?
Update: как найти нужный элемент по ID и изменить одно поле?
Delete: мягкое удаление (status: "deleted") vs жёсткое (убрать из массива)
Что лучше и почему?

2. Как сделать поиск и фильтрацию?

Поиск на клиенте vs на сервере — почему мы делаем на клиенте?
Array.filter() + String.includes() — как комбинировать поиск и фильтр?
Что если 1000 объявлений — будет тормозить?
Debounce: не искать на каждую букву, а ждать 300ms паузы

3. Как сделать «авторизацию» без бэкенда?

Никнейм в localStorage — почему это не настоящая безопасность?
Как проверить «это твоё объявление»?
Что мешает поменять никнейм в DevTools?
Зачем нужны настоящие системы авторизации?

4. Как структурировать данные?

Каждое объявление = объект с какими полями?
id — как генерировать? (Date.now() + random)
Массив объявлений vs объект {id: data} — что удобнее для CRUD?
Как не потерять данные при одновременном редактировании двумя людьми?

5. SPA или много страниц?

Одна страница с показом/скрытием секций — проще
Несколько HTML-файлов + передача ID через URL — понятнее
Hash-роутинг: #list, #create, #item/123

function generateId() {

}

CRUD-функции
```undefined
// CREATE
async function createItem(newItem) {
  const data = await fetchAll();
  newItem.id = generateId();
  newItem.author = getUser();
  newItem.created = new Date().toISOString();
  newItem.views = 0;
  newItem.status = 'active';
  data.items.push(newItem);
  await saveAll(data);
  return newItem;
}

// UPDATE
async function updateItem(id, updates) {
  const data = await fetchAll();
  const index = data.items.findIndex(i => i.id === id);
  if (index === -1) throw new Error('Не найдено');
  if (data.items[index].author !== getUser()) throw new Error('Не твоё!');
  Object.assign(data.items[index], updates);
  await saveAll(data);
}

// DELETE
async function deleteItem(id) {
  const data = await fetchAll();
  const index = data.items.findIndex(i => i.id === id);
  if (data.items[index].author !== getUser()) throw new Error('Не твоё!');
  data.items.splice(index, 1); // жёсткое удаление
  await saveAll(data);
}

```
Поиск + фильтрация
```undefined
function filterItems(items, { search, category }) {
  return items.filter(item => {
    const matchSearch = !search || 
      item.title.toLowerCase().includes(search.toLowerCase());
    const matchCategory = !category || category === 'all' || 
      item.category === category;
    return matchSearch && matchCategory && item.status === 'active';
  });
}

```
