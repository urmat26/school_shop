# School Shop

Школьный маркетплейс — площадка для покупки, продажи и обмена вещами между учениками.

**Демо:** [school-shop-nine.vercel.app](https://school-shop-nine.vercel.app)

## Возможности

- Полный CRUD объявлений (создание, просмотр, редактирование, удаление)
- Поиск, фильтрация по категории, сортировка, пагинация
- Загрузка фото через ImgBB
- Избранное и счётчик просмотров
- Авторизация по никнейму (localStorage)
- Тёмная и светлая тема (переключатель в шапке)

## Быстрый старт

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/urmat26/school_shop.git
   cd school_shop
   ```

2. Создайте `config.js` из шаблона:
   ```bash
   cp config.example.js config.js
   ```

3. Заполните ключи в `config.js`:
   - [JSONBin.io](https://jsonbin.io) — хранение данных
   - [ImgBB](https://imgbb.com) — загрузка изображений

4. Запустите локально (любой статический сервер):
   ```bash
   npx serve .
   ```
   Или откройте `index.html` через Live Server в VS Code.

## Структура

```
├── index.html      — список объявлений
├── create.html     — создание / редактирование
├── item.html       — страница объявления
├── favorites.html  — избранное
├── api.js          — API, утилиты, карточки
├── auth.js         — авторизация
├── config.js       — ключи (не в git)
└── style.css       — стили
```

## Деплой на Vercel

1. Подключите репозиторий на [vercel.com](https://vercel.com)
2. Добавьте `config.js` с ключами (или используйте переменные окружения + сборку)
3. Деплой произойдёт автоматически

## Безопасность

> **Важно:** ключи API в клиентском JS видны всем пользователям. Это нормально для учебного проекта, но не для продакшена. Для реального приложения нужен backend-сервер.

- Не коммитьте `config.js` — он в `.gitignore`
- Используйте `config.example.js` как шаблон
